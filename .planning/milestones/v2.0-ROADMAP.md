## Phase Details

### Phase 3: Website Discovery
**Goal**: Companies without a website get verified URLs found automatically — domain candidates are generated, checked for DNS and HTTP validity, confirmed as the right business, and saved to the database; Yelp enrichment runs daily within API limits
**Depends on**: Phase 2 (v1.0 shipped; existing enrichWorker.ts and domainUtils.ts to be connected)
**Requirements**: WEB-01, WEB-02, WEB-03, WEB-04, YELP-01, YELP-02
**Success Criteria** (what must be TRUE):
  1. Running the website discovery worker against a sample of 100 companies without websites produces verified URLs for any that have a discoverable domain — no crashes, no infinite loops
  2. A saved URL in `companies.website` is always a live site that returned 2xx/3xx, is not a directory or parked page, and has a page title with at least 50% token overlap with the company name
  3. Yelp enrichment runs a daily batch, stops at 5,000 calls, persists its `lastRowId` to a progress JSON, and skips records where phone and image are already populated
  4. Running Yelp enrichment a second time after a kill resumes from the saved rowid rather than restarting from zero
**Plans**: 2 plans

Plans:
- [ ] 03-01-PLAN.md — Refactor enrichWorker.ts: domain guessing via domainUtils.ts as primary path, verified URL written to DB
- [ ] 03-02-PLAN.md — Harden enrichYelpContacts.ts: SIGINT save, per-record lastRowId advance, pre-call skip guard

### Phase 4: Contact Extraction
**Goal**: Companies with a verified website get email addresses and phone numbers extracted by a real Playwright browser that navigates homepage and contact pages, handles JS-rendered content gracefully, and writes results directly to the database
**Depends on**: Phase 3 (requires verified website URLs in companies.website)
**Requirements**: CONT-01, CONT-02, CONT-03, CONT-04, CONT-05
**Success Criteria** (what must be TRUE):
  1. Running the contact extractor against a verified company URL returns any email addresses found on the homepage HTML
  2. When no email is found on the homepage, the scraper automatically navigates to `/contact`, `/contact-us`, or equivalent and extracts email from that page
  3. When a page is blocked, JS-only with no navigable links, or has no contact page, the scraper logs the failure and moves to the next company without throwing an unhandled exception
  4. Extracted email is written to `companies.email` and extracted phone fills `companies.phone` only when the field is currently null or empty
**Plans**: 2 plans

Plans:
- [ ] 04-01-PLAN.md — Create contactExtractor.ts: Playwright browser, homepage email/phone extraction, progress persistence, dry-run
- [ ] 04-02-PLAN.md — Add contact-page fallback: /contact, /contact-us, /about, /about-us tried in sequence when homepage has no email

### Phase 5: Orchestration
**Goal**: A single orchestrator script ties both pipelines together — running them on a daily schedule, persisting progress so any pipeline can be killed and resumed safely, and producing a daily summary with actionable metrics
**Depends on**: Phase 3 and Phase 4 (both pipelines must be runnable standalone before orchestration)
**Requirements**: ORCH-01, ORCH-02, ORCH-03
**Success Criteria** (what must be TRUE):
  1. Running `npm run enrich` (or equivalent) triggers both the Yelp and website/contact pipelines in sequence without requiring manual intervention
  2. Killing the orchestrator mid-run and restarting it resumes each pipeline from its last saved position — no records are re-processed from the beginning
  3. After each daily run, a summary log entry exists showing: companies processed, fields filled (website/email/phone), errors encountered, and Yelp API calls used
**Plans**: 1 plan

Plans:
