# Plan 04-02: Frontend Auth UI & Favorites Sync

## Goal
AuthContext provider, login/logout UI, and favorites that sync to server when authenticated.

## Task 1: AuthContext + API client + AuthButton

### Add auth types to src/api/types.ts
```typescript
export interface AuthUser {
  userId: number;
  email: string;
  firstName: string;
  lastName: string;
  accountId: number;
  membershipLevel: string;
}
```

### Add auth API to src/api/api.ts
```typescript
export const AuthAPI = {
  getLoginUrl(): string { return '/api/auth/login'; },
  async getMe(): Promise<{ user: AuthUser | null }> { ... },
  async logout(): Promise<void> { ... },
};

export const FavoritesAPI = {
  async getAll(): Promise<string[]> { ... },
  async add(companyId: string): Promise<void> { ... },
  async remove(companyId: string): Promise<void> { ... },
  async sync(ids: string[]): Promise<void> { ... },
};
```

### Create src/contexts/AuthContext.tsx
AuthProvider wrapping the app:
- On mount: call AuthAPI.getMe() to check session
- Provides: { user, isAuthenticated, isLoading, login(), logout() }
- login() navigates to /api/auth/login (full page redirect)
- logout() calls AuthAPI.logout(), clears user state
- Check for `?login=success` URL param to refetch user on callback return

### Create src/components/AuthButton.tsx
Navbar auth button:
- Unauthenticated: "Sign In with HCC" button (gold outline, link to login)
- Authenticated: User avatar/initials + dropdown with name, email, "Sign Out"
- Loading: subtle skeleton/spinner
- Uses dark theme styling consistent with BuildBoard
