# Plan 04-01: Wild Apricot OAuth Backend & Favorites API

## Goal
Server-side OAuth2 flow with Wild Apricot, JWT session management, and authenticated favorites CRUD.

## Task 1: Install dependencies + WA service + auth middleware

### Install
```bash
npm install jose cookie-parser
npm install -D @types/cookie-parser
```

### Create server/services/wild-apricot.ts
Wild Apricot OAuth client — all WA communication is server-side (no CORS support).

Exports:
- `getLoginUrl(state: string): string` — builds WA OAuth login redirect URL
- `exchangeCode(code: string, redirectUri: string): Promise<WATokenResponse>` — POST to oauth.wildapricot.org/auth/token with Basic auth
- `getUserInfo(accessToken: string, accountId: number): Promise<WAUser>` — GET /v2/accounts/{id}/contacts/me
- `WATokenResponse` type: { access_token, refresh_token, expires_in, permissions }
- `WAUser` type: { Id, FirstName, LastName, Email, MembershipLevel, Status }

**Config from env:**
- WA_CLIENT_ID
- WA_CLIENT_SECRET
- WA_ACCOUNT_URL (e.g., https://hcc.wildapricot.org)
- JWT_SECRET (for signing session tokens)

### Create server/middleware/auth.ts
JWT middleware using `jose`.

Exports:
- `optionalAuth(req, res, next)` — extracts JWT from `bb_session` cookie, sets `req.user` if valid, continues if not (never blocks)
- `requireAuth(req, res, next)` — same but returns 401 if no valid token
- `createSessionToken(payload: UserPayload): Promise<string>` — sign JWT with HS256, 7-day expiry
- `UserPayload` type: { userId: number, email: string, firstName: string, lastName: string, accountId: number, membershipLevel: string }

Declare `req.user` on Express Request via module augmentation.

## Task 2: Auth routes + favorites routes + server wiring

### Create server/routes/auth.ts
OAuth endpoints:

- `GET /api/auth/login` — generates random state, stores in cookie, redirects to WA OAuth
- `GET /api/auth/callback` — validates state, exchanges code for tokens, gets user info, creates JWT, sets httpOnly cookie `bb_session` (secure, sameSite=lax, 7-day maxAge), redirects to `/?login=success`
- `GET /api/auth/me` — uses optionalAuth, returns user payload or { user: null }
- `POST /api/auth/logout` — clears bb_session cookie, returns { success: true }

**Cookie settings:**
```typescript
res.cookie('bb_session', token, {
  httpOnly: true,
  secure: process.env.NODE_ENV === 'production',
  sameSite: 'lax',
  maxAge: 7 * 24 * 60 * 60 * 1000, // 7 days
  path: '/',
});
```

### Create favorites table in schema
Add to server/db/schema.ts (or inline in a migration):
```sql
CREATE TABLE IF NOT EXISTS user_favorites (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  userId INTEGER NOT NULL,
  companyId TEXT NOT NULL,
  createdAt TEXT DEFAULT (datetime('now')),
  UNIQUE(userId, companyId)
);
```

### Create server/routes/favorites.ts
Authenticated favorites CRUD:

- `GET /api/favorites` — requireAuth, returns user's favorite company IDs
- `POST /api/favorites/:companyId` — requireAuth, adds favorite (INSERT OR IGNORE)
- `DELETE /api/favorites/:companyId` — requireAuth, removes favorite
- `POST /api/favorites/sync` — requireAuth, accepts { ids: string[] } from localStorage, merges into DB (for migration on first login)
