**Step 1: Create server/services/gemini-search.ts**

Gemini function calling service for natural language query parsing.

Key components:

1. **Function declaration** (`SEARCH_FUNCTION`):
   - Name: `search_directory`
   - Parameters: query (STRING), category (STRING), city (STRING), state (STRING), minRating (NUMBER), services (ARRAY of STRING), summary (STRING, required)
   - Description: Include representative category names and instructions for the AI

2. **`aiParseQuery(message, history)`**:
   - Creates a GoogleGenAI instance with GEMINI_API_KEY
   - Builds conversation contents from history + system preamble
   - Calls `ai.models.generateContent()` with model `gemini-2.5-flash`
   - Uses `FunctionCallingConfigMode.ANY` to force function calling
   - Extracts filters and summary from the function call response
   - If matchCategory is available (from Plan 01), normalizes the category

3. **`smartSearch(message, history)`**:
   - Wraps aiParseQuery in a try/catch with 3-second Promise.race timeout
   - On success: returns `{ filters, summary, source: 'ai' }`
   - On failure: falls back to `parseChat(message)`, maps ChatParsed to SearchFilters
   - Fallback summary: generic "Here are results for your search" text
   - Returns `{ filters, summary, source: 'fallback' }`

4. **`isAIAvailable()`**:
   - Returns true if GEMINI_API_KEY is set in environment
   - Log warning on startup if key is missing

Handle edge cases:
- Empty/whitespace message → return empty filters with generic summary
- Gemini returns no function call → throw to trigger fallback
- API key missing → immediate fallback (don't attempt API call)
