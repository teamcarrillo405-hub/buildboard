**Step 1: Create server/services/fts5.ts**

FTS5 index management and search execution using better-sqlite3.

Key functions:
- `ensureFtsIndex()`: Check if `companies_fts` virtual table exists. If not, create it with external-content pointing at `companies` table. Create sync triggers. Run `INSERT INTO companies_fts(companies_fts) VALUES('rebuild')` to populate. Log progress. This runs once on first startup (~23 seconds).
- `ftsSearch(query, filters, limit, offset)`: Execute hybrid search combining FTS5 MATCH (for text) with SQL WHERE (for structured filters like state, category, rating). Return results with BM25 relevance scores.
- `sanitizeFtsQuery(input)`: Remove FTS5 syntax operators (quotes, parens, AND/OR/NOT/NEAR). Wrap individual words in double quotes. Add prefix matching on last word for type-ahead.

FTS5 table definition:
```sql
CREATE VIRTUAL TABLE IF NOT EXISTS companies_fts USING fts5(
  businessName, category, city, state, services,
  content='companies',
  content_rowid='rowid',
  tokenize='porter unicode61'
);
```

BM25 column weights: businessName=10, category=5, city=3, state=1, services=2.

Use `sqlite` (the raw better-sqlite3 instance) from `server/db.ts` for all FTS5 operations since Drizzle ORM does not support FTS5 virtual tables directly.

Search interface:
```typescript
interface SearchFilters {
  query?: string;      // Free-text for FTS5 MATCH
  category?: string;   // Exact category match
  state?: string;      // 2-letter state code
  city?: string;       // City name (case-insensitive)
  minRating?: number;  // Minimum star rating
  services?: string[]; // Service keywords
}

interface SearchResult {
  companies: Company[];
  totalResults: number;
  searchTime: number;   // ms
  source: 'fts5' | 'like'; // Which search method was used
}
```

**Step 2: Create server/helpers/categoryMatcher.ts**

Fuzzy category matching that maps AI-returned or user-typed categories to actual database values.

Functions:
- `matchCategory(input)`: Takes a category string (potentially from AI or user input) and returns the best matching category from the database. Uses this priority:
  1. Exact match (case-insensitive)
  2. Check CATEGORY_SYNONYMS from parseChat.ts
  3. Prefix match (e.g., "Plumbing" matches "Plumbing Contractor")
  4. Check CATEGORY_TO_IMAGE_SLUG keys for normalized group matching
  5. Return null if no match
- `getValidCategories()`: Returns the full list of distinct categories from the database (cached).
