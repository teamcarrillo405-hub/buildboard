# Plan 05-01: Verification Backend

## Goal
Add verification status to the database, admin API to manage it, and search ranking boost for verified businesses.


## Task 2: Admin API + search boost

### Create server/routes/admin.ts
Admin endpoints (requireAuth + admin check):

For v1, admin check is simple: check if user email is in ADMIN_EMAILS env var (comma-separated list).

- `GET /api/admin/companies?q=&status=&limit=&offset=` — search companies with verification status filter
- `PUT /api/admin/companies/:id/verification` — body: { status: 'unverified'|'verified'|'hcc_member' }
- `GET /api/admin/stats` — returns counts by verification status

### Admin middleware
Create `isAdmin` middleware in `server/middleware/auth.ts`:
```typescript
export function isAdmin(req, res, next) {
  if (!req.user) return res.status(401).json({ error: 'Authentication required' });
  const adminEmails = (process.env.ADMIN_EMAILS || '').split(',').map(e => e.trim().toLowerCase());
  if (!adminEmails.includes(req.user.email.toLowerCase())) {
    return res.status(403).json({ error: 'Admin access required' });
  }
  next();
}
```

### Update FTS5 search ranking
In `server/services/fts5.ts`, modify the search query to boost verified companies:
- Add verificationStatus to the SELECT
- Add a ranking modifier: verified companies get a 1.5x boost, hcc_member gets 2x
- Implementation: `ORDER BY (CASE WHEN c.verificationStatus = 'hcc_member' THEN rank * 0.5 WHEN c.verificationStatus = 'verified' THEN rank * 0.7 ELSE rank END)`
- BM25 returns negative scores (lower = better), so multiplying by <1 boosts
