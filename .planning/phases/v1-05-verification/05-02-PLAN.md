# Plan 05-02: Verification Frontend

## Goal
Display verification badges on company cards and profiles, and create an admin interface for managing verification status.

## Task 1: Badge component + types + CompanyCard/Profile updates

### Add verification types to src/api/types.ts
```typescript
export type VerificationStatus = 'unverified' | 'verified' | 'hcc_member';
```

Add `verificationStatus?: VerificationStatus` to the Company interface.

### Add admin API to src/api/api.ts
```typescript
export const AdminAPI = {
  async searchCompanies(params): Promise<...> { ... },
  async setVerification(id: string, status: VerificationStatus): Promise<void> { ... },
  async getStats(): Promise<{ unverified: number, verified: number, hcc_member: number }> { ... },
};
```

### Create src/components/VerificationBadge.tsx
Badge component showing verification status:
- `verified`: Gold shield/checkmark icon with "Verified" text
- `hcc_member`: Gold shield + "HCC Member" text with distinct HCC styling
- `unverified`: renders null (no badge)
- Two sizes: 'sm' (for CompanyCard) and 'md' (for CompanyProfile)
- Uses lucide-react icons: ShieldCheck, Award
- Styling: `bg-brand-gold/15 text-brand-gold border border-brand-gold/30 rounded-full px-2 py-0.5`
- HCC variant: `bg-brand-primary/15 text-brand-primary border border-brand-primary/30`

### Update src/components/CompanyCard.tsx
- Import VerificationBadge
- Show badge overlay on top-right of card image (position absolute)
- Small size ('sm')

### Update src/pages/CompanyProfile.tsx
- Import VerificationBadge
- Show badge next to company name in the header section
- Medium size ('md')

### Verify
- `npx tsc --noEmit` passes
- Commit: "feat(05-02): add VerificationBadge, display on cards and profiles"

## Task 2: Admin page + routing

### Create src/pages/Admin.tsx
Admin verification management page:
- Protected: check useAuth() and redirect if not admin
- Search bar to find companies
- Table/list view showing: company name, category, city/state, current status, action buttons
- Status dropdown to change verification: unverified → verified → hcc_member
- Stats header showing counts by status
- Dark theme consistent with BuildBoard
- Simple admin check: compare user email against hardcoded admin list or /api/admin/stats returns 403

### Update src/App.tsx routing
- Add `/admin` route pointing to Admin page
- Import lazy for code splitting

### Update Navbar
- If user is admin (email in admin list), show "Admin" link in navbar

- `npx tsc --noEmit && npm run build` passes
- Commit: "feat(05-02): add admin verification page and routing"
