  <files>server/scripts/orchestrator.ts</files>
  <action>
Create `server/scripts/orchestrator.ts` as a standalone TypeScript script (shebang: `#!/usr/bin/env npx tsx`).

The script must:

1. Record the run start time with `Date.now()`.

2. Read the "before" snapshot from each pipeline's progress JSON BEFORE spawning the pipeline. This enables computing the delta for this run (not lifetime totals). Helper: `readJsonSafe(path)` — returns parsed JSON or `{}` if file missing.

   Before-snapshots:
   - `before.yelp` = read `./logs/yelp_contact_enrich_progress.json` (or `{}`)
   - `before.web` = read `./logs/enrichWorker_0_of_1.json` (or `{}`)
   - `before.contact` = read `./logs/contact_extract_progress.json` (or `{}`)

3. Run each pipeline in sequence using `spawnSync` from Node.js `child_process`. Each pipeline must inherit the parent's stdio (pass `{ stdio: 'inherit', shell: false }`) so the user sees real-time output. Set no timeout — these are long-running jobs.

   Pipeline 1 — Yelp enrichment:
   ```typescript
   spawnSync('npx', ['tsx', 'server/scripts/enrichYelpContacts.ts'], { stdio: 'inherit', shell: false });
   ```

   Pipeline 2 — Web discovery:
   ```typescript
   spawnSync('npx', ['tsx', 'server/scripts/enrichWorker.ts', '--worker-id', '0', '--total-workers', '1'], { stdio: 'inherit', shell: false });
   ```

   Pipeline 3 — Contact extraction:
   ```typescript
   spawnSync('npx', ['tsx', 'server/scripts/contactExtractor.ts'], { stdio: 'inherit', shell: false });
   ```

   Log a separator line before each pipeline start:
   ```
   [orchestrator] ═══ Starting: enrichYelpContacts (1/3) ═══
   ```

4. After all three pipelines finish, read the "after" snapshot from each progress JSON.

5. Compute deltas (after - before) for each metric. Handle missing keys with `?? 0` to avoid NaN. This makes the summary reflect only what happened in THIS run, not lifetime totals.

   Yelp delta:
   ```typescript
   const yelpDelta = {
     apiCalls: (after.yelp.totalApiCalls ?? 0) - (before.yelp.totalApiCalls ?? 0),
     phoneFilled: (after.yelp.totalPhoneFilled ?? 0) - (before.yelp.totalPhoneFilled ?? 0),
     imageFilled: (after.yelp.totalImageFilled ?? 0) - (before.yelp.totalImageFilled ?? 0),
   };
   ```

   Web discovery delta:
   ```typescript
   const webDelta = {
     processed: (after.web.processed ?? 0) - (before.web.processed ?? 0),
     websitesFound: (after.web.found ?? 0) - (before.web.found ?? 0),
   };
   ```

   Contact extraction delta:
   ```typescript
   const contactDelta = {
     processed: (after.contact.totalSearched ?? 0) - (before.contact.totalSearched ?? 0),
     emailsFilled: (after.contact.totalEmailFilled ?? 0) - (before.contact.totalEmailFilled ?? 0),
     phonesFilled: (after.contact.totalPhoneFilled ?? 0) - (before.contact.totalPhoneFilled ?? 0),
   };
   ```

6. Compute total errors across all three pipelines:
   ```typescript
   const totalErrors =
     ((after.yelp.errors ?? 0) - (before.yelp.errors ?? 0)) +
     ((after.web.errors ?? 0) - (before.web.errors ?? 0)) +
     ((after.contact.errors ?? 0) - (before.contact.errors ?? 0));
   ```

7. Compute duration in human-readable format:
   ```typescript
   function formatDuration(ms: number): string {
     const totalSeconds = Math.floor(ms / 1000);
     const h = Math.floor(totalSeconds / 3600);
     const m = Math.floor((totalSeconds % 3600) / 60);
     const s = totalSeconds % 60;
     return h > 0 ? `${h}h ${m}m` : m > 0 ? `${m}m ${s}s` : `${s}s`;
   }
   ```

8. Build the summary entry:
   ```typescript
   const summary = {
     date: new Date().toISOString().slice(0, 10),  // "2026-03-18"
     yelp: yelpDelta,
     webDiscovery: webDelta,
     contactExtraction: contactDelta,
     duration: formatDuration(Date.now() - startMs),
     errors: totalErrors,
   };
   ```

9. Append summary to `logs/daily_summary.json`. This file is a JSON array. Read it (or start with `[]` if missing), push the new entry, then write it back with 2-space indent:
   ```typescript
   const SUMMARY_FILE = './logs/daily_summary.json';
   let existing: any[] = [];
   try { existing = JSON.parse(fs.readFileSync(SUMMARY_FILE, 'utf8')); } catch {}
   existing.push(summary);
   fs.writeFileSync(SUMMARY_FILE, JSON.stringify(existing, null, 2));
   ```

10. Print the summary to stdout in a readable format:
    ```
    [orchestrator] ═══ DAILY RUN COMPLETE ═══
    [orchestrator] Date     : 2026-03-18
    [orchestrator] Duration : 2h 14m
    [orchestrator] Yelp     : apiCalls=4800 phoneFilled=312 imageFilled=89
    [orchestrator] Web      : processed=500 websitesFound=67
    [orchestrator] Contacts : processed=300 emailsFilled=45 phonesFilled=12
    [orchestrator] Errors   : 3
    [orchestrator] Summary written to logs/daily_summary.json
    ```

Use only Node.js built-ins (`child_process`, `fs`, `path`) and `dotenv` for env loading. No new npm packages. The `logs/` directory must be created with `fs.mkdirSync('./logs', { recursive: true })` at the top of the script before any file reads.

Note: `spawnSync` blocks until the child process exits, which is exactly what we want — sequential execution. If the user kills the orchestrator between pipelines (CTRL+C on Windows sends SIGINT), the current pipeline's SIGINT handler saves its own progress and exits cleanly. On the next `npm run enrich`, the orchestrator re-reads the before-snapshots from the saved progress JSONs, so deltas will still be correct. (ORCH-02 is already handled by each pipeline's own progress persistence — the orchestrator does not re-implement resume logic.)
