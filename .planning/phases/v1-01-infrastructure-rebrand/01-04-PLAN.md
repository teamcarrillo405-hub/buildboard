<task type="auto">
  <name>Task 1: Configure Vercel deployment and deploy</name>
  <files>
    vercel.json

  </files>
  <action>
1. Create `vercel.json` at project root:
   ```json
   {
     "buildCommand": "npm run build",
     "outputDirectory": "dist",
     "rewrites": [
       { "source": "/api/(.*)", "destination": "/api" }
     ]
   }
   ```
   The rewrites rule sends all `/api/*` requests to the Express serverless function. The `outputDirectory` tells Vercel where the Vite build output lives for static file serving via CDN.

2. Create `api/index.ts` -- This is the entry point Vercel uses for the Express serverless function. Vercel looks for handlers in the `api/` directory:
   ```typescript
   export { default } from '../server/index.js';
   ```
   This is a simple re-export. The actual Express app (with all routes, middleware, db connection) is in `server/index.ts`.

   IMPORTANT: Verify that `server/index.ts` already has the conditional `app.listen()` guard (`if (process.env.VERCEL !== '1')`). This was included in Plan 01-01. Without it, the serverless function fails because Vercel provides its own listener.

3. Ensure the Vercel CLI is available:
   ```bash
   npx vercel --version
   ```
   If it prompts to install, accept.

4. Initialize git if not already done (Vercel works best with git):
   ```bash
   git status || (git init && git add -A && git commit -m "chore: initial commit for Vercel deployment")
   ```
   The `.gitignore` from Plan 01-01 prevents the 2.1GB database, node_modules, and secrets from being committed.

5. Deploy to Vercel (preview deployment first):
   ```bash
   npx vercel --yes
   ```
   During first run, it may prompt for:
   - Vercel account login (opens browser)
   - Project setup: accept defaults, or configure framework as "Vite"
   - Build command: `npm run build`
   - Output directory: `dist`

6. Set environment variables on Vercel for production and preview:
   ```bash
   npx vercel env add TURSO_DATABASE_URL production
   npx vercel env add TURSO_DATABASE_URL preview
   npx vercel env add TURSO_AUTH_TOKEN production
   npx vercel env add TURSO_AUTH_TOKEN preview
   ```
   Enter the values from `.env` (obtained in Plan 01-02) when prompted.

7. Redeploy to production with env vars active:
   ```bash
   npx vercel --prod
   ```

8. Test the production deployment:
   ```bash
   # Get the deployment URL
   DEPLOY_URL=$(npx vercel inspect --json 2>/dev/null | node -e "let d='';process.stdin.on('data',c=>d+=c);process.stdin.on('end',()=>{try{console.log(JSON.parse(d).url)}catch{console.log('check-manually')}})")

   # Test API
   curl -s "https://$DEPLOY_URL/api/stats"

   # Test frontend
   curl -sI "https://$DEPLOY_URL/" | head -5
   ```

9. If API returns errors, debug:
   - Check Vercel function logs: `npx vercel logs $DEPLOY_URL`
   - Common issues:
     a. `api/index.ts` not found -> ensure `api/` directory exists in git
     b. Module resolution errors -> ensure `.js` extensions in all server imports
     c. "TURSO_DATABASE_URL is undefined" -> env vars not set for this deployment, re-add them
     d. `app.listen()` called in serverless context -> ensure the `VERCEL !== '1'` guard is in place

10. If the standard Express detection doesn't work and you need a custom build step, add to `package.json`:
    ```json
    "scripts": {
      "vercel-build": "tsc && vite build"
    }
    ```
    This is usually not needed but helps with edge cases.
