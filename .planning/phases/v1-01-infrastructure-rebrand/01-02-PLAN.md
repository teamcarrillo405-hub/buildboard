<task type="auto">
  <name>Task 1: Trim database, upload to Turso, and update schema</name>
  <files>server/schema.ts, server/routes/companies.ts, .env</files>
  <action>
1. Audit the database file size:
   ```bash
   ls -la server/constructflix.db
   ```
   Confirm it is ~2.09 GB (over Turso's 2GB --from-file limit).

2. Trim the database to get under 2GB. The dataSource column contains 'state_directory' for ALL 3.4M rows and importedAt is an identical timestamp for all rows -- both are pure waste:
   ```bash
   sqlite3 server/constructflix.db "ALTER TABLE companies DROP COLUMN dataSource;"
   sqlite3 server/constructflix.db "ALTER TABLE companies DROP COLUMN importedAt;"
   sqlite3 server/constructflix.db "VACUUM;"
   ```
   VACUUM rebuilds the entire B-tree structure and reclaims space from the dropped columns.

3. Check file size again:
   ```bash
   ls -la server/constructflix.db
   ```
   If still over 2GB, fall back to --from-dump strategy:
   ```bash
   sqlite3 server/constructflix.db .dump > /tmp/buildboard-dump.sql
   turso db create buildboard --from-dump /tmp/buildboard-dump.sql
   ```
   If under 2GB (expected), use --from-file:
   ```bash
   turso db create buildboard --from-file server/constructflix.db
   ```

4. Check if Turso CLI is installed: `turso --version`. If not installed:
   ```bash
   curl -sSfL https://get.tur.so/install.sh | bash
   ```
   Check if authenticated: `turso auth whoami`. If not, run `turso auth login` (opens browser for auth).

5. After database creation, get connection details:
   ```bash
   turso db show buildboard --url
   turso db tokens create buildboard
   ```

6. Create `.env` file from `.env.example` (created in Plan 01-01) with the actual Turso URL and auth token from step 5.
