<objective>
Create the category image generation pipeline: normalize 134 database categories into ~50-60 visual groups, build a Cloudflare R2 client for CDN-backed image storage, and create a batch script that generates 5 AI image variations per group using Imagen 4 Fast (~300 images, ~$6 total) and uploads them to R2.

Purpose: Every company listing needs a relevant image. This plan creates the baseline image library that serves as the second tier of the fallback chain (Google Places photo -> R2 category image -> default). Without this, 100% of companies that lack Google Places photos would show a generic placeholder.

Output: R2 service module, category normalization map, batch generation script. After running the script, ~300 WebP images are stored in R2 at `categories/{slug}/{1-5}.webp`.
</objective>

**Step 2: Create `server/data/category-map.ts`.**

This file exports three things:

1. `CATEGORY_TO_IMAGE_SLUG: Record<string, string>` -- Maps every raw database category string (all 134+) to a normalized image slug. Apply these normalization rules:
   - Case normalize: "Plumbing" = "plumbing"
   - Suffix merge: "Plumbing Contractor" = "Plumbing Contractors" = "Plumbing" -> `plumbing`
   - Compound merge: "Plumbing and HVAC Contractors" -> `plumbing` (use primary trade)
   - Subtype to parent: "Kitchen remodeling contractors" -> `residential-remodelers`
   - Activity to trade: "Foundation repair" -> `foundation`
   - Any category without a clear mapping gets a reasonable group (never leave unmapped)

2. `IMAGE_SLUGS: string[]` -- Array of unique slug values derived from the map (the ~50-60 groups).

3. `CATEGORY_PROMPTS: Record<string, string>` -- Maps each image slug to a Gemini prompt. Each prompt should follow this pattern:
   ```
   "Professional [trade description] at a [setting], [specific action], photorealistic, high quality, bright natural lighting, 16:9 aspect ratio, no text or logos"
   ```
   Examples:
   - `plumbing`: "Professional plumber installing copper pipes in a modern commercial building, detailed pipework visible, photorealistic, high quality, bright natural lighting, 16:9 aspect ratio, no text or logos"
   - `electrical`: "Licensed electrician working on a commercial electrical panel with organized wiring, photorealistic, high quality, bright natural lighting, 16:9 aspect ratio, no text or logos"
   - `roofing`: "Roofing crew installing shingles on a large commercial building, aerial perspective, photorealistic, high quality, bright natural lighting, 16:9 aspect ratio, no text or logos"

   Write prompts for ALL ~50-60 slugs. Focus on professional B2B construction imagery -- these appear on company cards and profile heroes.

<task type="auto">
  <name>Task 2: Create batch image generation script</name>
  <files>server/scripts/generate-category-images.ts</files>
  <action>
Create `server/scripts/generate-category-images.ts` -- a standalone script run via `npx tsx server/scripts/generate-category-images.ts`.

**Behavior:**
1. Import IMAGE_SLUGS and CATEGORY_PROMPTS from `../data/category-map.js`
2. Import uploadToR2 from `../services/r2.js`
3. Import GoogleGenAI from `@google/genai`
4. Import sharp for image optimization

**Flow for each slug (5 variations):**
1. Call Imagen 4 Fast (`imagen-4.0-fast-generate-001`) via `@google/genai` SDK:
   ```typescript
   const ai = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });
   const response = await ai.models.generateImages({
     model: 'imagen-4.0-fast-generate-001',
     prompt: CATEGORY_PROMPTS[slug],
     config: { numberOfImages: 1 },
   });
   const imageData = response.generatedImages[0].image.imageBytes;
   const buffer = Buffer.from(imageData, 'base64');
   ```
2. Resize and optimize with sharp: `.resize(800, 450, { fit: 'cover' }).webp({ quality: 80 })`
3. Upload to R2 at key `categories/${slug}/${variation}.webp` with contentType `image/webp`
4. Log progress: `[12/60] Uploaded: categories/plumbing/3.webp`

**Concurrency control:**
- Process slugs sequentially (one at a time) to avoid Gemini rate limits
- Within each slug, generate 5 images sequentially (API may reject concurrent requests)
- Add a 1-second delay between image generations to respect rate limits

**Error handling:**
- If a single image fails, log the error and continue (do not abort the entire batch)
- Track and report failures at the end: "Generated 295/300 images. 5 failures: [list]"
- Support resuming: before generating, check if the R2 key already exists by attempting a HeadObject. If it exists, skip it. This allows re-running the script after partial failure.

**CLI features:**
- `--dry-run` flag: prints what would be generated without calling APIs
- `--slug plumbing` flag: generate images for a single slug only (useful for testing)
- Parse args from process.argv manually (no external CLI library needed)

**Environment requirements:**
- GEMINI_API_KEY (for Imagen 4 Fast)
- R2_ACCOUNT_ID, R2_ACCESS_KEY_ID, R2_SECRET_ACCESS_KEY, R2_BUCKET_NAME, R2_PUBLIC_URL (for R2 upload)
- Validate all env vars are present before starting, exit with clear error message if missing.

Also install the required packages if not already present:
```bash
npm install @google/genai @aws-sdk/client-s3 sharp
npm install -D @types/sharp
```
