```typescript
export const EnrichmentAPI = {
  async getEnrichment(companyId: string): Promise<EnrichmentData> {
    return fetchJSON<EnrichmentData>(
      `${API_BASE}/companies/${encodeURIComponent(companyId)}/enrichment`
    );
  },

  getPhotoUrl(photoName: string, maxWidth: number = 400): string {
    return `${API_BASE}/places/photo?name=${encodeURIComponent(photoName)}&maxWidth=${maxWidth}`;
  },
};
```

**Step 3: Create src/components/CompanyImage.tsx.**

A reusable image component implementing the 3-tier fallback chain:
1. **Google Places photo** (if `googlePhotoUrl` prop provided) -- uses `/api/places/photo` proxy
2. **R2 category image** -- uses `R2_PUBLIC_URL/categories/{slug}/{variation}.webp`
3. **Hardcoded default** -- a generic construction image URL

Props:
```typescript
interface CompanyImageProps {
  company: { id: string; category: string; googlePhotoUrl?: string | null };
  className?: string;
  width?: number;
  variant?: 'card' | 'hero'; // card = 400px, hero = 800px
}
```

Implementation details:

- **`getCategoryImageUrl(category, index)`**: Export this function. It normalizes the category to a slug using the same logic as `CATEGORY_TO_IMAGE_SLUG` in server/data/category-map.ts. Since we cannot import server code into the client, embed a lightweight slugification function:
  ```typescript
  function categoryToSlug(category: string): string {
    return category.toLowerCase()
      .replace(/\s+contractors?$/i, '')
      .replace(/\s+/g, '-')
      .replace(/[^a-z0-9-]/g, '');
  }
  ```
  Then construct: `${R2_PUBLIC_URL}/categories/${slug}/${(hashCode(company.id) % 5) + 1}.webp`

  For `R2_PUBLIC_URL`: read from `import.meta.env.VITE_R2_PUBLIC_URL` or fall back to empty string. Use a simple hash of company.id to deterministically pick variation 1-5 so the same company always shows the same image.

- **Fallback chain via useState + onError:**
  - Start with Google photo URL if available, else R2 category image
  - On error from Google photo, try R2 category image
  - On error from R2 category image, try hardcoded default
  - On error from default, render a colored div with the category initial (pure CSS fallback, never broken)

- Add `loading="lazy"` attribute for performance.
- Add `alt={company.businessName || 'Construction company'}`.
- Use Tailwind classes from props.className, default to `w-full aspect-video object-cover`.

For the hero variant (used on profile pages), request larger images: maxWidth=800 for Google photos.
