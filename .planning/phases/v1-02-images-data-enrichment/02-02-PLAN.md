**Step 2: Create server/services/google-places.ts.**

This module wraps the Google Places API (New) with two key functions using the two-tier request strategy from the research:

```typescript
const PLACES_API_BASE = 'https://places.googleapis.com/v1';
```

**`findPlaceId(businessName, city, state): Promise<PlaceSearchResult | null>`**
- Calls Text Search (New) at POST `${PLACES_API_BASE}/places:searchText`
- Headers: `Content-Type: application/json`, `X-Goog-Api-Key: process.env.GOOGLE_PLACES_API_KEY`, `X-Goog-FieldMask: places.id,places.displayName,places.formattedAddress`
- Body: `{ textQuery: "${businessName} ${city} ${state}", pageSize: 1 }`
- This is Essentials ID Only tier ($10/1K). DO NOT add fields like `places.photos`, `places.rating`, or `places.reviews` to the field mask -- those trigger Enterprise billing ($35/1K).
- Returns `{ placeId, displayName, formattedAddress }` or null if no results

**`getPlaceDetails(placeId): Promise<PlaceDetailsResult>`**
- Calls Place Details (New) at GET `${PLACES_API_BASE}/places/${placeId}`
- Headers: `X-Goog-Api-Key`, `X-Goog-FieldMask: photos,reviews,rating,userRatingCount,regularOpeningHours,websiteUri,nationalPhoneNumber`
- This is Enterprise + Atmosphere tier ($20/1K) because it includes `reviews`
- Returns typed result with photos array, reviews array, rating, hours, etc.

**Type definitions (export these):**

```typescript
}
```

**Step 1: Create server/routes/enrichment.ts.**

Single endpoint: `GET /companies/:id/enrichment`

Flow:
1. Look up company by ID from `companies` table. Return 404 if not found.
2. Check `googlePlacesCache` for existing place_id (query by companyId).
3. **Cache miss (no place_id):**
   - Call `findPlaceId(company.businessName, company.city ?? '', company.state ?? '')` from google-places service
   - If found, INSERT into `googlePlacesCache` with `onConflictDoUpdate` (upsert pattern):
     ```typescript
     await db.insert(googlePlacesCache).values({
       companyId,
       placeId: result.placeId,
       matchConfidence: 0.8,
       createdAt: new Date(),
       lastAccessedAt: new Date(),
     }).onConflictDoUpdate({
       target: googlePlacesCache.companyId,
       set: { placeId: result.placeId, lastAccessedAt: new Date() },
     });
     ```
   - If not found, return `{ enriched: false, placeId: null }`
4. **Cache hit (place_id exists):**
   - Update `lastAccessedAt` timestamp
   - Skip Text Search (saved ~$0.01)
5. **Fetch fresh details** (whether cache hit or miss, if we have a placeId):
   - Call `getPlaceDetails(placeId)` from google-places service
   - Return response shaped as:
     ```json
     {
       "enriched": true,
       "placeId": "ChIJ...",
       "photos": [{ "name": "places/xxx/photos/yyy", "widthPx": 4032, "heightPx": 3024, "authorAttributions": [...] }],
       "reviews": [{ "authorAttribution": {...}, "rating": 5, "text": {...}, "relativePublishTimeDescription": "a month ago" }],
       "rating": 4.7,
       "userRatingCount": 142,
       "regularOpeningHours": { "openNow": true, "weekdayDescriptions": ["Monday: 8:00 AM - 5:00 PM", ...] },
       "websiteUri": "https://...",
       "nationalPhoneNumber": "(555) 123-4567"
     }
     ```
   - Limit photos to first 5, reviews to first 5

**Step 2: Create server/routes/images.ts.**

Single endpoint: `GET /places/photo`

Query params:
- `name` (required): Google Places photo resource name (e.g., "places/xxx/photos/yyy")
- `maxWidth` (optional, default "400"): maximum width in pixels

Flow:
1. Validate `name` param exists, return 400 if missing
2. Validate GOOGLE_PLACES_API_KEY exists, return 503 if missing
3. Construct URL: `https://places.googleapis.com/v1/${name}/media?maxWidthPx=${maxWidth}&key=${GOOGLE_PLACES_API_KEY}`
4. Fetch with `redirect: 'follow'` (Google redirects to actual image)
5. If response not ok, return the error status
6. Set response headers:
   - `Content-Type` from upstream response
   - `Cache-Control: public, max-age=3600` (browser caches for 1 hour -- reduces repeat requests within session)
7. Stream the image buffer to the client

This is the critical security boundary: the API key stays on the server, the client only sees `/api/places/photo?name=...`
