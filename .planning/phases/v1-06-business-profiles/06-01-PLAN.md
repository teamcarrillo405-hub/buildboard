# Plan 06-01: Profile Editing Backend

## Goal
Backend API for verified businesses to edit profile fields and upload photos/videos via presigned R2 URLs.

## Task 1: Install presigner + create profile routes

### Install dependency
```bash
npm install @aws-sdk/s3-request-presigner
```

### Add presigned URL function to server/services/r2.ts
Import `getSignedUrl` from `@aws-sdk/s3-request-presigner` and `PutObjectCommand` (already imported).
Add function:
```typescript
export async function getPresignedUploadUrl(
  key: string,
  contentType: string,
  expiresIn = 900, // 15 minutes
): Promise<string> {
  const command = new PutObjectCommand({
    Bucket: R2_BUCKET,
    Key: key,
    ContentType: contentType,
  });
  return getSignedUrl(r2Client, command, { expiresIn });
}
```

### Create company_media table migration
Create `server/services/media.ts`:
- `ensureMediaTable()` — CREATE TABLE IF NOT EXISTS company_media (id TEXT PRIMARY KEY, companyId TEXT NOT NULL, type TEXT NOT NULL, r2Key TEXT NOT NULL, url TEXT NOT NULL, filename TEXT, fileSize INTEGER, sortOrder INTEGER DEFAULT 0, uploadedBy INTEGER, createdAt TEXT DEFAULT (datetime('now')))
- `addMedia(companyId, type, r2Key, url, filename, fileSize, uploadedBy): MediaRecord`
- `getMedia(companyId): MediaRecord[]` — ordered by sortOrder, type
- `deleteMedia(mediaId, companyId): boolean` — ensures companyId matches for safety
- `countMedia(companyId, type): number` — for limit checking

### Create server/routes/profile.ts
Profile editing endpoints (all require `requireAuth` + verification check inline):

**Verification check helper** (inline, not middleware):
```typescript
function canEditCompany(req, companyId): { allowed: boolean; reason?: string }
// Check: req.user exists, fetch company, check verificationStatus is 'verified' or 'hcc_member'
```

Endpoints:
- `PUT /api/profile/:id` — update editable fields (phone, email, website, services, hours, address, zipCode, warranty, emergencyService, freeEstimate)
  - Body: `{ field: value }` partial update
  - Validate: only allowed fields, sanitize input
  - Uses raw sqlite for UPDATE (build SET clause dynamically)
  - Returns updated company row

- `POST /api/profile/:id/upload-url` — generate presigned PUT URL
  - Body: `{ filename: string, contentType: string, fileSize: number }`
  - Validate: contentType in allowed list, fileSize under limit
  - Photo: max 10MB, image/jpeg|image/png|image/webp
  - Video: max 100MB, video/mp4|video/webm
  - Key format: `profiles/{companyId}/{uuid}.{ext}`
  - Returns: `{ uploadUrl, key, publicUrl }`

- `POST /api/profile/:id/media` — register uploaded media
  - Body: `{ key: string, type: 'photo'|'video', filename: string, fileSize: number }`
  - Check: count limits (20 photos, 5 videos)
  - Saves to company_media table
  - Returns: the media record

- `GET /api/profile/:id/media` — list company media (public, no auth needed)
  - Returns array of media records

- `DELETE /api/profile/:id/media/:mediaId` — remove media
  - Requires auth + verification check
  - Deletes from company_media (does NOT delete from R2 — orphan cleanup is a v2 concern)
  - Returns: `{ success: true }`

### Add ProfileAPI to src/api/api.ts
```typescript
export const ProfileAPI = {
  async update(companyId: string, data: Record<string, unknown>): Promise<Company> { ... },
  async getUploadUrl(companyId: string, file: { filename: string, contentType: string, fileSize: number }): Promise<{ uploadUrl: string, key: string, publicUrl: string }> { ... },
  async registerMedia(companyId: string, media: { key: string, type: string, filename: string, fileSize: number }): Promise<MediaRecord> { ... },
  async getMedia(companyId: string): Promise<MediaRecord[]> { ... },
  async deleteMedia(companyId: string, mediaId: string): Promise<void> { ... },
  async uploadFile(uploadUrl: string, file: File): Promise<void> {
    // Direct PUT to R2 presigned URL
    await fetch(uploadUrl, {
      method: 'PUT',
      body: file,
      headers: { 'Content-Type': file.type },
    });
  },
};
```

Add `profile: ProfileAPI` to the API export object.
