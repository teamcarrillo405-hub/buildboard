<objective>
Refactor enrichWorker.ts so that domain guessing via domainUtils.ts runs FIRST — before any
Bing/Google Playwright search — and only verified URLs (DNS + HTTP + title match) are saved.
The search engine fallback fires only when all generated candidates fail.

Purpose: Domain guessing is faster, free, and requires no Playwright browser spin-up. Connecting
the already-built domainUtils.ts to enrichWorker.ts eliminates redundant search engine calls for
companies that have discoverable domains.

Output: A refactored enrichWorker.ts where enrichBusiness() tries generateCandidateDomains first,
verifies each candidate, and returns on the first verified match. Search engine fallback remains
intact as the fallback path.
</objective>

    ```
    STEP 1 — Domain guessing (fast, no browser):
      candidates = generateCandidateDomains(businessName, city, state, category)
      for each candidate domain:
        if isDirectoryUrl('https://' + candidate): skip
        dns = await checkDomainDNS(candidate)
        if !dns: continue
        { valid, finalUrl } = await verifyWebsite('https://' + candidate)
        if !valid: continue
        { match } = await verifyBusinessMatch(finalUrl, businessName)
        if !match: continue
        // SUCCESS — return immediately, no search engine needed
        return { website: finalUrl, email: null, source: 'domain-guess' }

      if empty: candidates = await searchGoogle(baseQuery)

    STEP 3 — Nothing found:
      return { website: null, email: null, source: 'none' }
    ```
